# AQE v3.8.8 Benchmark Baseline

**Date**: 2026-03-25
**Environment**: Linux 6.12.76-linuxkit (Codespace)
**Node**: v22.x | Vitest 4.1.1

## How to reproduce

```bash
# Default (1K/10K tiers — CI-safe)
npx vitest bench tests/performance/ruvector-scale-benchmarks.bench.ts
npx vitest bench tests/performance/ruvector-native-hnsw.bench.ts
npx vitest bench tests/performance/ruvector-migration-benchmark.bench.ts

# Full scale (100K/500K tiers — local only, ~2GB RAM)
RUN_SCALE_BENCH=1 npx vitest bench tests/performance/ruvector-scale-benchmarks.bench.ts
RUN_SCALE_BENCH=1 npx vitest bench tests/performance/ruvector-migration-benchmark.bench.ts
```

## Comparison Guide

To compare against this baseline after changes:

1. Run benchmarks on the same hardware
2. Compare ops/sec and mean values
3. Key regressions to watch:
   - Insert throughput dropping >20%
   - Search latency increasing at same tier
   - Migration time exceeding extrapolated targets
   - Compression overhead growing beyond 15x baseline
