# v3.8.13 Release Notes

**Release Date:** 2026-03-30

## Highlights

New code intelligence CLI commands (`complexity`, `--incremental`, `--git-since`) give users full CLI parity with MCP tools. Security fix for command injection vulnerability in `--git-since`. Comprehensive documentation cleanup across 20 files fixing stale references.

## Added

- `aqe code complexity <path>` — Analyze cyclomatic, cognitive, and Halstead complexity metrics with hotspot detection and JSON output support
- `aqe code index --incremental` — Incremental indexing that only re-indexes changed files
- `aqe code index --git-since <ref>` — Git-aware indexing that only indexes files changed since a given ref (e.g., `HEAD~5`, `main`)
- Code intelligence CLI section added to README with all 7 command examples
- `--depth` validation with clear error message for invalid values

## Fixed

- **Security (CWE-78):** Command injection vulnerability in `--git-since` — user input was interpolated into shell command via `execSync`. Now uses `execFileSync` with array arguments
- **Runtime crash:** Missing `return` after `cleanupAndExit()` in complexity action could cause null-pointer dereference
- **20 files of stale references:** Non-existent `aqe kg` commands replaced with `aqe code`, phantom agent names replaced with actual agents, `ruflo doctor --fix` replaced with `aqe health`/`aqe init`
- `aqe init --auto` now correctly sets up MCP server by default
- Tool-scoping test data aligned with source — `hypergraph_query` added to all 5 scoped roles

## Changed

- Complexity analysis uses batched `Promise.all` (batch size 8) instead of sequential file processing
- `SOURCE_EXTENSIONS` exported from shared module to prevent divergence between scan paths
