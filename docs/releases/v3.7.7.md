# v3.7.7 Release Notes

**Release Date:** 2026-03-02

## Highlights

Four new browser capabilities inspired by Scrapling's adaptive scraping patterns, ported to TypeScript for the QE fleet: adaptive locators that survive UI changes, resource blocking for faster E2E tests, a stealth browser client for bot-protected environments, and a browser page pool for parallel execution.

## Added

- **Resource blocking** with 3 presets (`functional`, `visual`, `performance`) and domain-based tracker/ad detection for 30+ known domains. Blocks non-essential resources during E2E tests for faster page loads.
- **Adaptive locator service** that falls back to text, ARIA, and fingerprint similarity matching when primary selectors break. Uses weighted scoring (tag, role, classes, text, attributes, DOM position) with configurable similarity threshold.
- **Browser page pool** for managing concurrent page lifecycle with async acquire/release, health checks, and idle pruning. Waiters queue when pool is exhausted and get notified on release.
- **Stealth browser client** via Patchright (optional, lazy-loaded) for bot-protected test environments. Supports persistent contexts, Cloudflare challenge wait, proxy config, and integrated resource blocking.
- **`stealth-testing` use case** and `'stealth'` tool preference in the browser client factory.

## Changed

- Click and type step executors now use adaptive locator fallback chain when configured.
- Resource blocking is applied during browser orchestrator launch for agent-browser clients.
- `getBrowserToolAvailability()` now includes stealth availability.

## Security

- Fixed potential JS injection in adaptive locator text fallback (replaced manual escaping with `JSON.stringify()`).
