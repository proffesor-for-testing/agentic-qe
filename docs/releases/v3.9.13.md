# v3.9.13 Release Notes

**Release Date:** 2026-04-17

## Highlights

ADR-093 Opus 4.7 migration: Sonnet 4.6 becomes the fleet-wide default, `xhigh` effort is on everywhere, Opus 4.7 is the opt-in escalation target — zero fleet breakage at the June 15 Sonnet 4 retirement.

## Added

- **Opus 4.7, Sonnet 4.6, Haiku 4.5 in the model registry** with capability flags for adaptive thinking, `xhigh` effort, tokenizer version, and 1M context.
- **`xhigh` effort level, fleet-wide** — every qe-* agent runs at `xhigh` by default; override per-agent via frontmatter, per-run via `QE_EFFORT_LEVEL`, or globally via `config/fleet-defaults.yaml`.
- **Cyber Verification Program application draft** at `docs/security/cyber-verification-application.md` covering `qe-pentest-validator`, `qe-security-scanner`, `qe-security-auditor`, `qe-security-reviewer` — these remain pinned to Sonnet 4.6 until enrollment clears.
- **Opus 4.7 as the default ADR-092 advisor model** for vendor-agnostic escalation paths.

## Changed

- **`DEFAULT_SONNET_MODEL` constant replaces hardcoded model IDs** across routing, providers, cost tracker, and CLI. Tier 2 and Tier 3 both route to Sonnet 4.6; Opus 4.7 is reserved as an opt-in escalation target via `MultiModelExecutor`.
- **Post-audit sweep** — CI workflow, consensus provider, MCP sampling handler, 12 skill evals, 9 test files, opencode plugin, scripts, `.kiro`, and command docs updated to the new model lineup.

## Fixed

- **Zero fleet breakage at 2026-06-15 Sonnet 4 retirement** — every live routing reference to `claude-sonnet-4-20250514` has been removed. Remaining mentions are intentional compatibility metadata (deprecation dates, Bedrock mappings, historical cost entries).
- **User-facing model listings** now reflect the new model lineup end-to-end.
