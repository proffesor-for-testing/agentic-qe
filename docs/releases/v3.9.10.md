# v3.9.10 Release Notes

**Release Date:** 2026-04-13

## Highlights

Provider-agnostic advisor strategy (ADR-092) adds multi-model routing with circuit breaking, PII redaction, and domain-specific prompts — letting the Queen coordinator distribute advisory tasks across different LLM providers safely and efficiently.

## Added

- **Provider-agnostic advisor subsystem** (`src/routing/advisor/`) — Multi-model executor, circuit breaker, PII redaction, domain prompts, and typed interfaces for advisor configuration
- **`aqe llm-router` CLI command** — Inspect and manage advisor routing configuration
- **MCP protocol-server advisor integration** — Advisor tools available as MCP tools for fleet agent access
- **Shared executor preamble** — Common frontmatter for QE agents using the advisor subsystem
- **8 QE agents updated** — coverage-specialist, fleet-commander, pentest-validator, queen-coordinator, risk-assessor, root-cause-analyzer, security-auditor, test-architect now advisor-aware
- **ADR-092 documentation** with Phase 0 trial results and statistical analysis scripts

## Fixed

- Distribution pipeline edge cases in multi-model executor
- Error handling tightened in routing-feedback integration

## Changed

- Queen task management integrates advisor routing for delegated tasks
- Tiny-dancer router extended with advisor fallback path
- Routing feedback collects advisor response quality signals
