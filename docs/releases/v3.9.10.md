# v3.9.10 Release Notes

**Release Date:** 2026-04-13

## Highlights

**Use any LLM provider for QE advisory tasks — not just the one running your session.** Route simple questions to cheaper models, keep complex reasoning on premium providers, and let the fleet automatically route around provider outages. Sensitive data is scrubbed before any prompt leaves your environment.

## Why this matters

Until now, all QE agent reasoning happened through a single LLM provider per session. That meant:
- If your provider had an outage, your entire fleet stalled
- Simple advisory tasks ("what test should I write next?") cost the same as complex analysis
- You couldn't mix providers (e.g., Ollama for local work, Claude for hard problems)

v3.9.10 solves all three. The Queen coordinator can now distribute advisory questions across multiple providers based on task complexity, cost, and provider health.

## What you get

- **Lower costs** — Route simple advisory tasks to cheaper or self-hosted models while keeping complex reasoning on premium providers
- **Higher availability** — Per-provider circuit breakers detect degradation and reroute automatically; no more fleet-wide stalls from a single provider outage
- **Data safety by default** — API keys, credentials, and PII are automatically stripped from prompts before they reach any external advisor. Three modes: `strict` (redacts credentials + PII), `balanced` (credentials only), `off` (for self-hosted providers like Ollama)
- **8 QE agents ready out of the box** — coverage-specialist, fleet-commander, pentest-validator, queen-coordinator, risk-assessor, root-cause-analyzer, security-auditor, and test-architect all support multi-provider routing immediately
- **`aqe llm-router` CLI command** — Check which providers are configured, their health status, circuit breaker state, and routing weights

## Getting started

```bash
npx agentic-qe@3.9.10 init --auto
aqe llm-router status
```

## Technical details

- New `src/routing/advisor/` module: multi-model executor, circuit breaker (closed/open/half-open), PII redaction pipeline, domain-specific prompt templates
- 65 new tests across 5 test files
