# v3.8.9 Release Notes

**Release Date:** 2026-03-25

## Highlights

Multi-language coverage analysis expands to 6 new formats (JaCoCo, dotcover, Tarpaulin, Go cover, Kover, xcresult), with security hardening of the witness chain and production-scale RuVector benchmarks.

## Added

- **Multi-language coverage parsers** — 6 new parsers: JaCoCo (Java/Kotlin), dotcover (C#/.NET), Tarpaulin (Rust), Go cover, Kover (Kotlin/JVM), and xcresult (Swift/iOS). Coverage analysis now works natively with polyglot projects.
- **Language-aware agent routing** — The agent routing layer now accepts a `language` parameter, enabling language-specific model selection and strategy. MCP schemas updated with the new field.
- **RuVector P1 scale benchmarks** — Production-scale benchmarks for 10K and 100K vector operations, concurrent access stress tests, and HNSW memory usage reporting via `aqe ruvector status`.
- **Knowledge graph language extensions** — Swift and C# file extension mappings added to the code intelligence knowledge graph for broader polyglot support.

## Fixed

- **SHA-256 witness hashing** — Replaced the insecure djb2 hash with SHA-256 in the witness adapter, ensuring cryptographic integrity of coherence proofs.
- **Coherence gate witness persistence** — Witnesses are now correctly persisted to the proof envelope, maintaining audit trail continuity across sessions.
- **Hardcoded signing key removed** — Eliminated a hardcoded key from the coherence gate; signing keys are now derived from configuration.
- **Benchmark output formatting** — Fixed RuVector benchmark result display to correctly report metrics.
