# v3.7.22 Release Notes

**Release Date:** 2026-03-14

## Highlights

Fixes hook path resolution that broke when Claude Code ran hooks from a different working directory, prevents SQLite corruption with safe database opens everywhere, and removes all v2 migration code.

## Fixed

- **Hook path resolution (#352)** — Helper scripts (`brain-checkpoint.cjs`, `statusline-v3.cjs`) used `process.cwd()` to find the project root, which broke when Claude Code ran hooks from a different working directory. Now uses `path.resolve(__dirname, '..', '..')` for reliable resolution regardless of `cwd`.
- **Invalid JSON in settings.json** — Hook commands for `SessionStart`, `Stop`, and `UserPromptSubmit` had unescaped double quotes around `$(git rev-parse ...)` subshells, producing invalid JSON that Claude Code could not parse.
- **Pattern growth pipeline unblocked** — Pattern promotion and metrics queries referenced the removed `learning_experiences` table. Updated to use `captured_experiences` with correct column mappings.
- **SQLite corruption prevention (#348)** — All database open calls now use the safe wrapper which sets WAL mode, `busy_timeout=5000`, and `foreign_keys=ON` consistently.

## Changed

- **V2 migration code removed** — The `aqe migrate` CLI command, V2-to-V3 migration wizard, and all supporting code (~2,400 lines) have been removed. No v2 installations exist in the wild.
- **README updated** — Removed the V2 to V3 migration section.
