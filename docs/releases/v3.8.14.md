# v3.8.14 Release Notes

**Release Date:** 2026-03-31

## Highlights

Security and reliability patch: fixes SQL injection in witness-chain, removes ~6 MB `@faker-js/faker` runtime dependency from generators, and resolves 3 P0 blockers from QE v3.8.13 audit (dead MCP server, CI test gate, ESLint).

## Fixed

- **SQL injection in witness-chain LIMIT/OFFSET** — Parameterized query bindings replace string interpolation. Offset-without-limit now works correctly. `limit=0` returns zero rows instead of all rows.
- **`@faker-js/faker` removed from 7 generator files** — Replaced with lightweight `test-value-helpers.ts` (node:crypto only). Generators no longer crash when devDependencies are absent.
- **`aqe init` hook paths break from subfolders** — Uses `CLAUDE_PROJECT_DIR` for reliable path resolution.
- **ruflo permissions leaked into user settings** — `aqe init` now only injects AQE-specific permissions.
- **Dead MCP `server.ts` (911 lines)** — Removed unused server to eliminate dual-server divergence risk.
- **CI published without test gate** — `npm-publish.yml` now requires all unit tests to pass before publish.
- **ESLint broken** — Renamed `.eslintrc.js` to `.eslintrc.cjs` for ESM compatibility.
- **Hardcoded version `3.0.0` in MCP servers** — Now reads dynamically from `package.json`.
- **Vitest process hang** — Worker-level teardown for native modules (`better-sqlite3`, `hnswlib-node`).

## Added

- `test-value-helpers.ts` — Zero-dependency test data generator using `node:crypto`.
- Pagination edge case tests for witness-chain (limit=0, offset-without-limit).
- 17 unit tests for test-value-helpers covering boundaries and inverted ranges.
