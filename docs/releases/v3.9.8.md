# v3.9.8 Release Notes

**Release Date:** 2026-04-08

## TL;DR

**Release-process maturity release.** No source code changes since v3.9.7 — every commit lands in CI infrastructure, fixtures, scripts, or documentation. The published package is functionally identical to v3.9.7 except for a refreshed lockfile carrying two transitive security patches. The value of this release is the structural improvements *around* the package: codeload-drift insurance for the release gate ([#411](https://github.com/proffesor-for-testing/agentic-qe/issues/411)), CI enforcement of the failure-modes PR template checkbox ([#408](https://github.com/proffesor-for-testing/agentic-qe/issues/408)), a weekly chaos workflow for adversarial-rare init failures ([#410](https://github.com/proffesor-for-testing/agentic-qe/issues/410)), and human-readable release verification docs ([#409](https://github.com/proffesor-for-testing/agentic-qe/issues/409)). Together these close 4 of the 5 #401 follow-up issues.

## Highlights

- **Codeload-drift insurance** ([#411](https://github.com/proffesor-for-testing/agentic-qe/issues/411)) — Release gate keeps working when GitHub regenerates `git archive` output. Three corpus tarballs now have a self-hosted mirror at the [`init-corpus-v1`](https://github.com/proffesor-for-testing/agentic-qe/releases/tag/init-corpus-v1) release; `setup.sh` falls back to the mirror on primary-URL sha256 mismatch and emits a loud warning.
- **PR template enforcement** ([#408](https://github.com/proffesor-for-testing/agentic-qe/issues/408)) — The honor-system #401 failure-modes checkbox is now checked by CI on every PR. 11 forbidden-phrase patterns ("I believe it's unlikely", "highly unlikely", "can't see how this would fail", etc.) hard-fail unless accompanied by a same-paragraph tracking issue reference. Dependabot allowlisted.
- **Weekly chaos workflow** ([#410](https://github.com/proffesor-for-testing/agentic-qe/issues/410)) — 6 pathological project shapes (UTF-16LE BOM, mixed line endings, symlink loop, binary-as-text, minified single-line bundle, control characters) generated at runtime and exercised against `aqe init` every Sunday. Asserts the watchdog catches hangs, not whether init succeeds on garbage input.
- **`docs/VERIFICATION.md`** ([#409](https://github.com/proffesor-for-testing/agentic-qe/issues/409)) — Maintainer-facing entry point for the post-#401 verification layer with architecture diagram, failure interpretation guide, and the procedure for adding fixtures. `docs/policies/release-verification.md` rewritten around the automated gate (the pre-#401 manual checklist is removed; the gate replaces it).
- **Verification matrix in release notes** ([#409](https://github.com/proffesor-for-testing/agentic-qe/issues/409)) — New `scripts/embed-verification-matrix.sh` parses the gate's `summary.txt` artifact from any workflow run and emits a markdown matrix for inclusion in per-version release notes. This is the first release that ships the script; the matrix below is generated from this release's gate run.
- **Transitive security patches** ([#414](https://github.com/proffesor-for-testing/agentic-qe/pull/414)) — `hono` 4.12.9→4.12.12 and `@hono/node-server` 1.19.11→1.19.13 via dependabot's `npm_and_yarn` security group.

## What changed for users

If you `npm install agentic-qe@3.9.8`:

- **Same code as v3.9.7.** No new features, no behavior changes, no bug fixes in the published package itself.
- **Refreshed lockfile** with the two transitive security patches above. Matters for downstream `npm ci` workflows.
- **Better release-process guarantees going forward.** Future releases benefit from drift insurance, enforced PR discipline, weekly adversarial coverage, and visible verification reporting — but those benefits accrue starting *after* this release ships, not in the package itself.

## What changed for maintainers

- The release gate is now resilient to a class of GitHub infrastructure failures it couldn't survive before.
- PR review for failure-mode reasoning is no longer honor-system. After this release lands, promote `Lint PR body` and `Self-test parser against fixtures` to required status checks in branch protection (Settings → Branches → main).
- A new weekly workflow exists for chaos coverage. Optionally smoke-test it with `gh workflow run init-chaos.yml --ref main` after the release lands; otherwise it fires Sunday 05:00 UTC.
- Per-release verification matrices can now be embedded into release notes via `./scripts/embed-verification-matrix.sh <run-id> >> docs/releases/vX.Y.Z.md` after `npm-publish.yml` completes for a tag.
