# v3.10.2 Release Notes

**Release Date:** 2026-06-02

## Highlights

Self-learning hardening release. Fixes two user-reported bugs that quietly broke
pattern learning — **#508** (the post-task hook recorded every Task trajectory as
`success=0` / `agent='unknown'`, so patterns never formed) and **#509** (the dream
scheduler's SessionStart readout was perpetually stale) — and adds importance-weighted
forgetting protection, contradiction detection, and real verification harnesses for
the learning and MCP subsystems.

## Fixed

- **#508 — Task trajectories silently recorded as failed.** The generated
  PostToolUse hook (`post-task … --success --json`) declared `--success` as a
  value-taking option, so Commander ate `--json` as the success value: every
  trajectory got `success=0`, and with no `--agent` passed, `agent='unknown'`.
  Distillation groups by agent/domain/success, so trajectories were unpromotable
  and pattern counts stayed near zero. `--success` is now optional-valued and the
  hook passes real agent attribution + description. **Re-run `aqe init`** on
  existing installs to pick up the corrected hook.
- **#509 — Stale dream-scheduler readout.** Post ADR-094, the
  `dream-scheduler:hook-state` row SessionStart reads was only incremented by the
  hook and never reset by the kernel scheduler that actually dreams, so
  `lastDreamTime` stayed `null` and pending-experiences grew unbounded. The kernel
  `DreamScheduler` now reconciles that row after each successful cycle.
- **MCP unknown-tool errors** now return a proper JSON-RPC `-32601` instead of
  `Internal error: [object Object]`.
- **Failure-based pattern deprecation** now fires — the row mapper dropped the
  `consecutive_failures` column it depended on.
- **Routing confidence** no longer washed out by broad domain detection (the
  domain-match denominator is capped; recovers confidence on multi-domain tasks,
  ranking preserved, tunable).

## Added

- **EWC++ importance-weighted forgetting protection** — proven high-value patterns
  resist decay/stale-deprecation; reversible, low-importance patterns unaffected.
- **Contradiction detection in consolidation** — same-context/opposite-outcome
  experiences suppress the loser from retrieval instead of merging.
- **Self-learning verification harness** (`npm run benchmark:self-learning`).
- **Real MCP protocol + tool-parity smokes** (`npm run mcp:validate`,
  `npm run mcp:parity`) — replaces the previous no-op `mcp:validate`.
- **Resilient hook shim** (`.claude/hooks/aqe-hook.sh`).
- **Pretrain-from-history bootstrap** (`npm run pretrain:history`).
- **RaBitQ 1-bit signature utility** (`npm run benchmark:rabitq`) — 32× index-memory
  reduction (utility only; not wired into a retrieval path yet).
- **Plugin contract smoke** (`npm run plugin:smoke`).
- **ADR-098** — learning-store & routing hardening decisions.

## Changed

- **`aqe init` installs a resilient hook shim and routes hooks through it.** A
  cross-platform Node shim (`.claude/hooks/aqe-hook.cjs`) is copied into each
  project and generated hooks run `node .../aqe-hook.cjs <cmd>` instead of
  `npx agentic-qe hooks`. It strips init noise from the transcript, never blocks
  a turn, prefers the project-local install (falls back to `npx`), and re-init
  migrates existing `npx`-based hooks (no duplicates).
- Router state-encoding regression guard added (audit of two upstream router bugs,
  confirmed not present in AQE).
