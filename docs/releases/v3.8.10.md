# v3.8.10 Release Notes

**Release Date:** 2026-03-26

## Highlights

Fixes the coverage data pipeline so that quality scores reflect real test coverage instead of fabricated numbers. Previously, `quality_assess` could report up to 95% coverage for files with zero tests.

## Fixed

- **Coverage data pipeline** — Unified 5 fragmented key conventions into a consistent `coverage:latest`, `coverage:previous`, and `coverage:file:{path}` ecosystem. Quality scores now use real coverage data or report "unavailable" — never fabricated values.
- **Test runner coverage collection** — `test_execute_parallel` now passes `--coverage` flags to vitest/jest and reads `coverage-summary.json` from disk when coverage isn't embedded in JSON output.
- **Per-file coverage storage** — Coverage is stored via key-value API alongside vector store, so quality-analyzer, defect-predictor, and defect-investigation all read real per-file data.
- **Coverage trend detection** — `coverage:previous` is now rotated before each new snapshot, enabling quality-gate trend detection that was always returning "stable".
- **Quality score fairness** — Missing coverage reports as -1 (unavailable) rather than 0%, avoiding a false 25-point penalty for projects without coverage tooling.
- **Coverage tracker worker** — Reads real data from `coverage:latest` and gap patterns from the gap-detector service instead of returning hardcoded fake values.
- **Orphaned key cleanup** — Removed dead writes to `quality-assessment:coverage:{hash}` and dead reads from `coverage-analysis:file:{path}` across coordinator, plugin, defect-predictor, and defect-investigation modules.

## Files Changed

12 files across 7 domains: quality-assessment, test-execution, coverage-analysis, defect-intelligence, coordination protocols, workers, and their tests.
