# v3.10.0 Release Notes

**Release Date:** 2026-05-20

## Highlights

Optional **external embedder endpoint** for AQE's semantic vector layer
(#503, ADR-097). When `AQE_EMBEDDER_ENDPOINT` is set, AQE skips loading
`@huggingface/transformers` in-process and routes feature-extraction through
an OpenAI-compatible `/v1/embeddings` service — eliminating duplicate model
loads in `ruflo`/`ruvector` co-deployments and per-hook cold-load latency
(15ms cold against a local `llama-server` vs ~1s for the in-process cold
load). Default unset = no behavior change. Followed by a hardening pass that
closed 11 findings from a pre-merge devil's-advocate audit, including two
silent-corruption paths that would have poisoned the HNSW index on a flaky
endpoint.

## Added

- **External embedder endpoint** (#503, ADR-097). Set
  `AQE_EMBEDDER_ENDPOINT=http://host:port` (or `unix:/path/to.sock`, or
  `EmbeddingConfig.endpoint`) and AQE routes vector computation off-process.
  - **OpenAI wire format** with `encoding_format: 'float'` pinned. Verified
    end-to-end against `llama-server` serving `all-MiniLM-L6-v2.Q8_0.gguf`;
    the same shape is what TEI / vLLM / Ollama / LocalAI / LM Studio /
    OpenAI advertise, so they should work as drop-in, but each is
    unverified until a real-provider integration test lands per provider.
  - **HTTP + HTTP-over-Unix-socket** transports.
  - **Startup probe** + stable identity fingerprint of a canary embedding,
    persisted to `memory.db` `kv_store` (`_system` namespace) so cross-run
    drift fires a loud warning on the next boot.
  - **Circuit breaker** (3 failures / 60s). Recovery invalidates the cached
    identity so the next embed re-probes — endpoint restarts often coincide
    with model swaps.
  - **TLS knobs** (`ca`, `cert`, `key`, `rejectUnauthorized`, `servername`)
    for self-hosted HTTPS without `NODE_TLS_REJECT_UNAUTHORIZED=0`.
  - **Bearer auth** via `AQE_EMBEDDER_TOKEN` env. URL userinfo is stripped
    and warned (env-only convention enforced).
  - **No silent hash fallback** — hard-fail with circuit breaker is the only
    correctness-preserving behavior at a vector-store boundary.
- **ADR-097** — External Embedder Endpoint (OpenAI-Compatible). 196-line
  decision record covering the wire-format choice, hard-fail rationale,
  identity fingerprint scheme, co-deployment topologies (dedicated
  embedder daemon / ruvector hosts / ruflo hosts), and a per-caller audit
  table showing all 12 embed-callers are non-poisoning under endpoint mode.
- `tests/integration/embedder-endpoint-llamacpp.test.ts` — six live tests
  against a real `llama-server`, gated on `AQE_LLAMA_EMBED_URL` so it skips
  cleanly in CI.

## Fixed

- **`sqlite-persistence.ts` backfill no longer poisons the HNSW index.**
  Previously, on endpoint failure, the backfill loop substituted
  `hashEmbedding()` for the remaining patterns — writing non-comparable
  vectors next to transformer vectors in the same index. Now re-raises and
  aborts the batch under `isUsingEndpoint()`. The in-process hash fallback
  is preserved for ARM64 ONNX compatibility.
- **`qe-reasoning-bank.embed()` propagates errors under endpoint mode.**
  The fail-soft hash fallback continues to work for the in-process path.
- **True TCP-connect timeout.** Connect-phase timeout fires at the socket
  layer (`socket.setTimeout` on the `socket` event, then phase switches to
  `requestTimeoutMs` on connect). Previously `req.setTimeout` was used,
  which is an idle-after-write timeout — so SYN-dropped endpoints hung for
  the OS default ~75s instead of the documented 5s.
- **Identity probe is gated on `embed()`.** First `embed()` lazily runs
  `probe()` with in-flight dedup (20+ concurrent embeds share one probe).
  Closes a path where a direct client construction could bypass the
  dim/identity boundary check.
- **Endpoint URL userinfo redacted from logs.** Credentials passed via URL
  (against the env-only convention) are stripped at parse and warned.

## Changed

- Embedding cache key is now namespaced by mode
  (`endpoint:<fingerprint>:<text>` vs `inproc:<text>`). Mode-flipping within
  a process can no longer return a vector computed under the wrong mode.
- Identity is persisted in `memory.db` `kv_store` (unified persistence)
  rather than a sidecar file under `.agentic-qe/`. This matches the project
  rule that all state goes through the single SQLite database.

## Numbers

- **Cold path against real `llama-server`** (localhost, MiniLM Q8_0):
  import 15.5 ms + init+probe+embed 15.2 ms = **30.7 ms** end-to-end.
- **Warm embed** (keep-alive socket, cached identity): **1.6 ms**.
- **33 unit tests** for the endpoint client (up from 23).
- **6 live integration tests** against a real provider.

## Migration

No migration required when `AQE_EMBEDDER_ENDPOINT` is unset (default).

To opt in:

```bash
# Run any OpenAI-compatible /v1/embeddings server. Example with llama.cpp:
llama-server -m all-MiniLM-L6-v2.Q8_0.gguf \
  --port 8080 --embeddings --pooling mean -c 512

# Point AQE at it:
export AQE_EMBEDDER_ENDPOINT=http://127.0.0.1:8080
# Or:
export AQE_EMBEDDER_ENDPOINT=unix:/run/embedder.sock
# Optional auth:
export AQE_EMBEDDER_TOKEN=your-token-here
```

Note: vectors written under one fingerprint are not comparable to vectors
written under a different fingerprint. If you switch models or quantizations,
the existing `.agentic-qe/patterns.rvf` HNSW index will warn loudly on next
boot but is **not** auto-rebuilt. Re-indexing is an explicit operator action.
