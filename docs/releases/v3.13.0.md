# v3.13.0 Release Notes

**Release Date:** 2026-07-18

## Highlights

**QE-Court** — an adversarial review court where a change is prosecuted by
independent AI reviewers from *different vendors*, and a `SHIP` verdict must
survive an escalating deeper review before it stands. It debuted by reviewing
this release and **blocked it**, catching three data-loss/crash bugs a green unit
suite had missed (all fixed). Also: a Codex (GPT) provider, a significance gate
for the self-learning flywheel, automatic `memory.db` recovery, and per-agent
cost tracking.

## Added

- **QE-Court (`/qe-court`).** Independent, cross-vendor reviewers (Codex/GPT,
  Cognitum, Claude) attack a diff/PR/test-suite with their own probe sets; a
  passing grade must *survive* an escalating overturn round before it stands.
  Three-valued verdict (`SHIP` / `REMAND` / `BLOCK`), signed court record, human
  as final judge. Rules enforced in code + a runnable acceptance eval
  (`aqe eval run --skill qe-court`) — trust tier 3.
- **Codex CLI provider.** Run QE on your ChatGPT subscription via `codex exec`
  (mirrors the `claude-code` subscription provider) — a cross-vendor GPT engine
  for review diversity and provider fallback.
- **`accept/v1+sig` flywheel significance gate.** Promotion now requires a
  *statistically significant* held-out gain (paired bootstrap CI > 0), so the
  self-learning loop no longer promotes within-noise improvements.
- **Automatic `memory.db` recovery.** A genuinely-corrupt learning database is
  restored from the newest integrity-verified backup automatically; the corrupt
  file is parked, never deleted. A healthy database is never touched.
- **Per-agent cost attribution** in the cross-process spend ledger.
- **Cognitum QE cost-Pareto benchmark** (`scripts/cognitum-qe-bench/`) — measures
  the cheapest model tier that clears the quality bar per QE task against
  `api.cognitum.one`. First finding: cheap tier matched/beat frontier at 3–4× less cost.
- Draft `/v1/qe/verdict` API spec for a future hosted QE-Court.

## Fixed

- **4 HIGH-severity CVEs no longer reach consumers (#565)** — `@huggingface/`
  `transformers` (→ `onnxruntime-node` → `adm-zip`) is now an optional peer
  dependency; a default install no longer pulls it. Local embeddings still work
  when present; endpoint embeddings unaffected.
- **Three data-loss/crash bugs QE-Court caught on its own first review** — an
  early subprocess exit could crash the caller (`EPIPE`); the new auto-restore
  could tear the DB under concurrency or misdiagnose a healthy DB on bind-mount
  filesystems (now atomic + locked + conservative); a failed ledger migration
  could silently stop recording spend and bypass the budget cap. Witness
  verification now fails closed; the flywheel gate rejects under-powered samples.

## Changed

- User-facing QE skill count is now **76** (adds `qe-court`); **50** Tier-3
  verified. `CONTRIBUTING.md` clarifies `ruflo` / `claude-flow` are development-time
  tooling, not runtime dependencies of the shipped package.
