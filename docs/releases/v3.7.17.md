# v3.7.17 Release Notes

**Release Date:** 2026-03-11

## Highlights

Six BMAD-inspired quality engineering improvements: adversarial review enforcement, agent customization overlays, structured validation pipelines, mechanical edge case detection, context compilation, and micro-file step architecture.

## Added

- **Adversarial review with minimum findings** — Review agents enforce a minimum weighted finding score (CRITICAL=3, HIGH=2, MEDIUM=1, LOW=0.5). Second-pass review with halved confidence runs when findings fall below threshold. Clean code requires explicit justification.
- **Agent customization overlays** — Customize agent behavior via `.claude/agent-overrides/*.yaml` with replace, append, and config sections. Integrated at init and runtime.
- **Structured validation pipelines** — 13-step requirements validation with blocking gates, weighted category scoring, and markdown reports.
- **Mechanical edge case detection** — Branch enumerator detecting 13 construct types via `aqe coverage gaps <file>`.
- **Context compilation for agents** — Parallel gathering from memory DB, git history, test files, and coverage data with token budget management.
- **Micro-file step architecture** — 5 QCSD skills split into 44 step files (<500 lines each).

## Changed

- QCSD skills restructured from monolithic (2000-2700 lines) to orchestrator + step files
- Agent definitions enhanced with minimum finding requirements and mechanical mode
- Review skills (brutal-honesty, sherlock, code-review-quality) now enforce minimum findings
- Deprecated v2 agents and legacy agent directories removed
- DevPod/devcontainer configuration refreshed

## Fixed

- Hooks phase installation for devcontainer compatibility
- Settings merge logic correction
