# v3.9.0 Release Notes

**Release Date:** 2026-04-02

## Highlights

Major production-hardening release with 11 CC-internals improvements, a 4-tier context compaction pipeline, extensible plugin architecture, and a background QE Quality Daemon — backed by 466+ new tests.

## Added

- **CC-internals improvements (IMP-00 through IMP-10)** — 11 improvements across 4 priority tiers, all integrated into production code paths with comprehensive test coverage.
- **4-tier context compaction pipeline** — Progressive memory compaction: microcompact (tier 1), session summary (tier 2), LLM compact (tier 3), reactive eviction (tier 4). Context budget tracking prevents overflows.
- **Plugin architecture** — Extensible plugin system with lifecycle management, manifest validation, dependency resolution, security sandboxing, and caching. Supports local, GitHub, and npm sources.
- **QE Quality Daemon** — Background quality monitoring with priority queue, git watcher, coverage delta analyzer, CI monitor, test suggester, nightly consolidation, notification service (SSRF-guarded), and persistent SQLite memory.
- **Retry engine** — Configurable retries with exponential backoff, jitter, circuit breaker, and per-error-class strategies.
- **Prompt cache latch** — Field-level caching controls to reduce redundant LLM calls.
- **Session durability middleware** — MCP session persistence and resume across connection drops.
- **Startup fast paths** — Parallel module prefetch and fast-path boot for unused subsystems.
- **Hook security hardening** — SSRF guard, config snapshot verification, standardized exit codes.
- **8 env var kill switches** — Granular feature control: `AQE_MICROCOMPACT`, `AQE_RETRY_DISABLED`, `AQE_SESSION_DURABILITY`, `AQE_PROMPT_CACHE_LATCH`, `AQE_FAST_PATHS`, `AQE_HOOKS_SSRF_DISABLED`, `AQE_COMPACTION_DISABLED`, `AQE_PLUGINS_DISABLED`.
- **`aqe daemon` and `aqe plugin` CLI commands** — Manage the QE daemon and plugins from the command line.

## Changed

- **Lazy-load CLI handlers** — Heavy imports deferred until command execution, reducing startup cost.
- **esbuild code splitting** — Chunk output with splitting for smaller initial bundles.
- **CLI index refactored** — Workflow command extracted, lazy-registry for on-demand loading.

## Fixed

- **Internal log leak to stdout** — Internal prefixed logs redirected to stderr.
