# v3.12.0 Release Notes

**Release Date:** 2026-07-10

## Highlights

A **learning-integrity layer** so the self-learning system can only promote what
it can prove. Test-quality is measured against a **frozen, hash-pinned oracle
benchmark**; promotions are gated by the *quality of their evidence* (real test
execution beats an LLM judge beats a structural heuristic), recorded as
**cryptographically-signed, replayable receipts**, and **re-checked against the
exact frozen rule** before they take effect. A new **two-gate quality command**
(mechanical checks + an independent frontier-model judge) is exposed on both the
CLI (`aqe quality-gate`) and MCP (`qe/quality/gate`). Ships with a
**designed-experiment tool** that measures — with real statistics — whether a
harness feature actually improves test quality or just adds cost. See ADRs
[117](../implementation/adrs/ADR-117-frozen-oracle-anchor-set.md)–[122](../implementation/adrs/ADR-122-designed-experiment-harness-gate.md).

## Added

- **`aqe quality-gate` CLI command and `qe/quality/gate` MCP tool** — a two-gate
  quality verdict: a mechanical gate (the generated test must actually run and
  kill mutations of a reference) plus an independent frontier-model judge that
  checks the test against the spec. Returns a three-valued verdict
  (pass / fail / inconclusive) and degrades honestly to "inconclusive" when the
  judge is unavailable rather than guessing.
- **Frozen oracle anchor set** — a hash-pinned benchmark of graded test items
  with a constant denominator, so a drop in the learning system's test-quality
  is detectable as a real regression instead of being masked by a moving target.
  Loading throws on any hash drift.
- **Provenance-tiered promotion** — every learning promotion now carries an
  evidence tier (`oracle:test-exec` > `judge:llm` > `proxy:structural`), and a
  policy can require a minimum tier before anything is promoted.
- **Receipt-gated policy flywheel** — retrieval-policy improvements are recorded
  as Ed25519-signed, replayable receipts with a compounding lineage, so every
  promotion is auditable, re-verifiable offline, and reversible. Honest nulls
  (no improvement) are a first-class, recorded outcome.
- **Gate re-execution on promotion paths** — before a promotion takes effect the
  frozen accept-rule is re-run on the sealed inputs, catching any promotion that
  was logged but that the rule would actually reject (a forged-promotion guard).
- **Designed-experiment harness tool** — `scripts/doe-run.ts` +
  `scripts/doe-aggregate.mjs` run a factorial screen (model × prompt × retrieval
  × scaffold) with replicates and a real main-effects ANOVA (F-test). The first
  screen found heavy scaffolding *lowers* reliability (p≈10⁻³⁸) and that
  retrieval adds no measurable quality on tasks the model already knows.

## Fixed

- **Brain-export hook no longer leaks** deadlocked background exports that could
  respawn, and database backups are hardened for virtiofs bind-mounts (a
  VACUUM-INTO backup path that avoids the WAL-corruption failure mode).
- **`AQE_DISABLE_WAL` / `AQE_JOURNAL_MODE` env override** for SQLite journal mode,
  so environments prone to bind-mount WAL corruption can opt into a safer mode.
- **Pattern-lifecycle promotion** now records and gates on provenance tier
  (additive column migration with a conservative backfill).

## Changed

- Nothing removed or renamed — this release is **additive**. Existing behavior is
  preserved; the new gates are opt-in and the provenance column migration is
  backward-compatible.
