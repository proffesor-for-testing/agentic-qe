# v3.10.6 Release Notes

**Release Date:** 2026-06-11

## Highlights

The Pattern Space cross-pollination release (#522): a QE review of a community
framework surfaced disciplines worth turning inward — evidence-labeled findings,
a real pass/fail safety eval for the data-protection rules, drift-proofing for
shipped agents, and benchmark honesty. Six ADRs (105–110), each with a live
verification record.

## Added

- **Evidence-class labels on findings (ADR-105)** — `EXECUTED` / `STATIC` /
  `INFERRED` / `CONJECTURE`. Gates block only on verified evidence; inference
  routes to adversarial verification first; the `validation_pipeline` MCP tool
  exposes the class + a `needsVerification` queue. All 51 standard `qe-*` agents
  carry an evidence-discipline contract.
- **Behavioral safety eval (ADR-106)** — the `memory.db` data-protection
  absolutes are now a pass/fail gate (deterministic engine + live model-judgment
  matrix across Claude haiku/sonnet/opus, opener and mid-task, 5/5 bar). All
  native tiers pass; the run caught and fixed a harness false-positive.
- **Shipped-agent invariant verifier + CI (ADR-107)** — `verify:invariants`
  asserts shipped agents keep their non-negotiable sections (content, not
  keywords) and haven't drifted from source; enforced on PR + release.
- **Benchmark lineage + pre-registered rubrics (ADR-108)** — a lineage registry
  and rubric-before-data discipline (rubric hash in every result row) so
  benchmark claims are auditable from the repo alone.
- **Interaction benchmark for qualitative agents (ADR-109)** — reactive
  simulated developer, hidden-test ground truth, non-same-family judge,
  scenario-clustered stats. Validated with a live cross-family run.
- **Kept-nulls in the learning loop (ADR-110)** — failed pattern applications
  are recorded as first-class negative records; retrieval surfaces them and
  discounts by context-matched failure density.

## Fixed

- `validation_pipeline` MCP projection now carries `evidenceClass` +
  `needsVerification` through the protocol boundary (was stripped).
- Learning engine now attributes guiding patterns and threads a context
  fingerprint into search, so failures attach to the right patterns.

## Changed

- Validation-pipeline gating is evidence-aware (verified evidence blocks;
  unverified inference warns and queues for verification).
- Database schema v10 adds the additive `qe_pattern_nulls` table (migration
  verified against a copy of the production DB; pre-existing counts unchanged).

## Notes

- ADR-105 `INFERRED` path is in place; wiring LLM-backed domain findings to
  emit it is follow-up. ADR-109's 2-scenario corpus is intentionally small —
  infrastructure complete, corpus scaling is follow-up.
