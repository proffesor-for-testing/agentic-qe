# v3.7.14 Release Notes

**Release Date:** 2026-03-08

## Highlights

Brain export upgraded to cover all 25 learning tables with streaming, witness chain v3 with Ed25519 signing, and npm packaging fix.

## Added

- **Brain Export v3.0 — full 25-table portable intelligence** — Completely rebuilt brain export/import to cover all 25 learning tables (up from 4). Data-driven `TABLE_CONFIGS` pattern replaces hardcoded per-table logic, with FK-aware ordering, Base64 BLOB serialization, and automatic DDL creation on import.
- **Streaming export for large tables** — Tables exceeding 10K rows now stream directly to JSONL using synchronous fd I/O, avoiding OOM on large databases.
- **Witness chain v3** — Dual hash algorithm support (SHA-256/SHAKE-256), 12 witness action types wired to production mutation sites, Ed25519 key persistence with PEM files.
- **Witness backfill CLI** — `aqe brain witness-backfill` replays existing learning data into the witness chain for older databases.
- **RVF adapter: freeze, derive, indexStats** — Immutability after signing, COW branching, HNSW index metrics.
- **RVF manifest sidecar** — `{path}.manifest.json` written alongside RVF exports.
- **Brain CLI lineage & signature display** — `aqe brain info` shows RVF lineage and signature status.

## Fixed

- **npm install ENOTEMPTY error** — Reduced package from 5,473 to 3,293 files.
- **ReDoS vulnerability in trigger-optimizer** — Replaced vulnerable regex.
- **Import atomicity** — Brain import wrapped in `db.transaction()`.
- **Domain collection in streaming path** — Fixed SQL syntax for large pattern tables.

## Changed

- Brain export manifest version bumped to `3.0` with per-table record counts.
- Merge engine supports 4 strategies: `latest-wins`, `highest-confidence`, `union`, `skip-conflicts`.
- ADRs updated: ADR-065, ADR-070, ADR-073.
