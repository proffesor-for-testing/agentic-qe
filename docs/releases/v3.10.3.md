# v3.10.3 Release Notes

**Release Date:** 2026-06-03

## Highlights

Code-intelligence now works on the **entire JS/TS ecosystem out of the box** —
`aqe code index` / `deps` / `impact` / `search` parse TypeScript and JavaScript
via bundled tree-sitter WASM grammars, with **no `typescript` dependency
required** (#511). This release also fixes a critical bug where **every MCP
domain tool hung forever on a fresh install**.

## Added

- **TS/JS code intelligence without the TypeScript compiler.** Bundled
  tree-sitter WASM grammars for `typescript` / `tsx` / `javascript` extract
  functions, classes/methods, interfaces and imports. Before, every TS/JS file
  errored with "TypeScript is required for code analysis" and produced 0
  nodes/edges unless the user separately installed the ~25 MB TypeScript
  compiler. The compiler is still used automatically as a higher-fidelity path
  when it happens to be installed. Grammars are reproducible via
  `scripts/build-grammars.sh` (pinned to the web-tree-sitter toolchain).

## Fixed

- **MCP domain tools hung indefinitely on a fresh install.** `code_index`,
  `quality_assess`, `defect_predict` and every other tool routed through the
  queen/router path spun a CPU core and never returned the first time pretrained
  patterns were seeded: `QEReasoningBank.initialize()` recursed through
  `storePattern()` because `initialized` was set *after* seeding. The CLI was
  unaffected. Fixed the init ordering, and bounded the advisory
  pattern-search/routing step with a timeout so any future stall there can never
  block a tool call.
- **`aqe code deps` returned 0 edges on TS/JS repos (#511).** `index` and
  `deps`/`search` run as separate CLI processes, but the knowledge graph was
  in-memory only — so `deps` always saw an empty graph. The graph is now
  persisted; relative imports (`./math`) resolve to their file nodes so the
  dependency graph connects; and directed traversal no longer fabricates phantom
  `A→B→A` cycles.
- **`aqe code search` returned 0 results (#511).** Indexed content was stored with
  a `{namespace}` option but read back without it (the #491 Bug 2 pattern), so
  every matched vector was dropped before returning.
- **CLI and MCP now share the code graph.** Pinned to a fixed
  `code-intelligence:kg` namespace (was the backend default, which differed
  between CLI and MCP), so the init existence check works and a graph indexed by
  either tool is visible to the other. `index --incremental` prunes stale
  entities/edges, the file-content cache is evicted before re-reads, and MCP
  `code_index` now reports real symbol/relation counts (was 0/0).
- **`index` warns on unsupported languages** instead of reporting edges that
  `deps` then shows as 0.
- **Repaired the bundled tree-sitter grammars** for python/java/c#/rust/swift,
  which were built with the old emscripten format and silently fell back to regex
  under web-tree-sitter 0.26.

## Changed

- **Lint cleanup** (1072 → ~472 problems): removed dead imports, wrapped
  case-clause declarations, documented intentional rule violations, ignored
  `src/_archived`, aligned bounded-context metadata. No behavioral changes.
