# v3.7.10 Release Notes

**Release Date:** 2026-03-05

## Highlights

Fixes three bugs reported after v3.7.9 upgrades: MCP server path resolution, CRLF line endings breaking skill parsing, and stale `v3/` directory references. Also rewrites the README for clarity.

## Fixed

- **MCP server entry path resolution** — `aqe mcp` command failed with "Could not find MCP server entry point" because `findMcpEntry()` went 2 levels up from `dist/cli/` instead of 1 to reach `dist/mcp/bundle.js`
- **Stale v3/ directory references in workers** — `10-workers.ts` and `mcp.ts` referenced `node_modules/agentic-qe/v3/dist/` which was removed in the v3.7.5 flatten; updated to `node_modules/agentic-qe/dist/`
- **CRLF line endings breaking skill frontmatter** — 170+ skill files (SKILL.md, schemas, evals, configs) had Windows-style `\r\n` endings causing `skill-lint` YAML parsers to fail on `---\r` delimiters
- **CI Node.js version** — All 8 CI workflow files upgraded from Node.js 20 to Node.js 24

## Changed

- **README rewritten** — Reduced from 1,097 to ~280 lines; now organized around user outcomes (what AQE does, quick start, platform support, usage examples) instead of version-by-version feature dumps and internal architecture details
- **Asset build safety** — `prepare-assets.sh` now normalizes CRLF→LF line endings and removes `.DS_Store` files as a safety net during npm publish preparation
