# v3.8.11 Release Notes

**Release Date:** 2026-03-27

## Highlights

Four new features from the Six Hats Tier 3 strategic investments, delivering deterministic YAML pipelines, heartbeat management, cost-optimized routing, and session-level token savings.

## Added

- **YAML Deterministic QE Pipelines (Imp-9)** — Declarative, token-free quality gates using YAML. Includes approval gate steps with auto-approve timeouts, 4 built-in SQL-only actions (quality-gate-check, coverage-threshold, pattern-health, routing-accuracy), and 7 CLI commands (`aqe pipeline load/validate/run/list/status/approve/reject`).

- **Heartbeat CLI & MCP Integration (Imp-10)** — Full user-facing access to the token-free heartbeat scheduler. 6 CLI commands (`aqe heartbeat status/run-now/history/log/pause/resume`), 3 MCP tools (`heartbeat_status`, `heartbeat_trigger`, `heartbeat_log`), history persistence, and timeout support.

- **Economic Routing Model (Imp-18)** — Quality-per-dollar tier scoring with budget-aware selection, cost-adjusted rewards for the neural router, and EMA-smoothed quality estimates. Opt-in via `enableEconomicRouting()`. CLI: `aqe routing economics/accuracy/metrics`. MCP: `routing_economics`.

- **Session Operation Cache (Imp-15)** — O(1) SHA-256 fingerprint-based cache for repeated operations. Runs before HNSW O(log n) similarity search, delivering 40-60% token savings on repeated tasks. Cross-session persistence via SQLite. CLI: `aqe token-usage --cache-stats`. MCP: `session_cache_stats`.

## Fixed

- Path traversal vulnerability in CLI `heartbeat log --date` command
- Approval gate timer/promise leak on workflow cancellation
- Economic routing config weight validation (clamp + normalize)
- In-place sort mutation in `selectTier()` fallback paths
- MCP `routing_economics` handler now initializes collector with persisted DB state
