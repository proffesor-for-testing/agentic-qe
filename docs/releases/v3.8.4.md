# v3.8.4 Release Notes

**Release Date:** 2026-03-19

## Highlights

Security hardening (P0 command injection + SQL injection fixes), learning database corruption repair, 30-40% smaller bundles, and CI parallelization cutting build times from 25+ min to ~15 min.

## Fixed

- **Command injection vulnerability (P0)** — Replaced `exec()` with `execFile()` + allowlist in test-verifier.ts to eliminate CWE-78 shell injection risk.
- **SQL injection surface (P0)** — Unified SQL table allowlists across sql-safety.ts, unified-memory.ts, and ruvector/brain-shared.ts; added `validateTableName()` to all 7 SQL interpolation sites.
- **70 unguarded process.exit() calls (P1)** — Replaced with return/throw in learning.ts (45) and hooks.ts (25) so cleanup handlers execute properly.
- **Database corruption** — Rebuilt corrupted memory.db (82 corrupt pages causing 43% dream cycle failures).
- **15K junk learning patterns** — Purged benchmark/test artifacts that inflated pattern counts and degraded quality scoring.
- **Benchmark test isolation** — Added `useUnified: false` to prevent benchmark runs from polluting the production learning database.
- **Statusline pattern inflation** — Fixed statusline to count only meaningful patterns.
- **CI timeouts** — Split monolithic CI jobs into parallel shards (Optimized CI: 6 jobs, MCP tests: 4 shards), wall-clock 25+ min to ~15 min.
- **CI concurrency groups** — PRs cancel stale runs; main branch runs no longer cancel each other.

## Changed

- **Bundle size reduced** — CLI 9.8 to 6.9 MB (-30%), MCP 12 to 7.2 MB (-40%) via minification.
- **@faker-js/faker moved to devDependencies** — ~8 MB savings from production install.
- **Verification scripts hardened** — Real checks with non-zero exit codes on failure.
- **4 perma-failing scheduled workflows disabled** — Manual dispatch still available.
- **Quality scoring improved** — Context-aware scoring replaces hardcoded values.

## Added

- **12 QE swarm analysis reports** — Comprehensive v3.8.3 audit covering security, performance, code quality, SFDIPOT, dependencies, API contracts, architecture, and accessibility.
