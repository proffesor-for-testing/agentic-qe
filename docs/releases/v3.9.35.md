# v3.9.35 Release Notes

**Release Date:** 2026-05-19

## Highlights

A test-infrastructure release. Restores ESM subpath imports of `HnswAdapter`
(PR #497), and starts dismantling the cumulative-growth pattern that has been
timing out fork workers on `qe-reasoning-bank.test.ts` and
`aqe-learning-engine.test.ts` across multiple recent publish workflows (issue
#448). No production-code behavior changes.

## Fixed

- **`import 'agentic-qe/kernel/hnsw-adapter'` threw `ERR_PACKAGE_PATH_NOT_EXPORTED`**
  (#497). The compiled file shipped in `dist/kernel/hnsw-adapter.js`, but the
  subpath was missing from the `exports` map, so Node ESM strict resolution
  refused the import even though the artifact was present. Adds the missing
  entry alongside `./kernel`.

- **Vitest fork workers timed out on the heaviest learning tests** (issue #448,
  steps 2 + 3). Two interventions land together:

  1. **Split the two largest test files into thematic forks.** With
     `pool=forks`, `maxForks=1`, `fileParallelism=false`, each test file
     gets its own sequential fork — so splitting a 1000+ line file caps
     per-fork heap.
     - `qe-reasoning-bank.test.ts` (1019 lines → 386 lines core) now has
       two companions: `qe-reasoning-bank-utilities.test.ts` (pure-function
       tests, no bank — runs in ~10 ms) and
       `qe-reasoning-bank-feedback-loop.test.ts` (heavy `recordOutcome`
       block isolated).
     - `aqe-learning-engine.test.ts` (622 lines → 366 lines core) gets
       `aqe-learning-engine-patterns.test.ts` (heaviest sub-suites) and
       `aqe-learning-engine-factory.test.ts` (constructor-only tests,
       no `initialize()` overhead).
     - Coverage preserved: every test still runs under `test:ci`, just
       distributed across three forks per group instead of one.

  2. **Release module singletons between tests so per-test memory growth
     doesn't compound across the file.** Two singletons in the transitive
     graph weren't being reset:
     - `real-embeddings.ts` had `clearEmbeddingCache()` and
       `resetInitialization()` exports that nothing was calling.
     - `audit/witness-chain.ts` retained its `_instance` (and the
       accumulated chain entries + better-sqlite3 handle) across tests
       with no reset path.
     - Adds a `_resetWitnessChainForTests()` helper next to the existing
       factory, and wires both resets into the heavy test files' `afterEach`
       (and `resetInitialization()` into `afterAll`).

## Changed

- **`test:unit:fast` exclude list** now also carries the two new heavy split
  files (`qe-reasoning-bank-feedback-loop.test.ts`,
  `aqe-learning-engine-patterns.test.ts`) so they don't accidentally run
  in the fast lane while the dispose-hook story is being validated by CI.
  The two cheap split files (`*-utilities.test.ts`, `*-factory.test.ts`)
  are *included* in the fast lane and restore coverage that was previously
  dark.

## Added

- **`tests/integration/package-exports.integration.test.ts`** — a contract
  test that pins each declared subpath in the `package.json#exports` map
  (entry exists, files exist on disk, documented symbol is reachable).
  Future regressions of PR #497's class fail this test instead of breaking
  consumers' imports silently.

## Internal

- Test scope tracking: `qe-reasoning-bank-rvf-backfill.test.ts` still has a
  `describe.skip` block (PR #445 regression coverage). Issue #448's
  acceptance criterion to un-skip stays open — the heavy memory work in
  this release lays the groundwork but doesn't itself close it.

## Verification

This release ships no production-code behavior changes. The kernel exports
fix is verified by the new contract test. The test-infrastructure changes
are verified by CI running the split files in their separate forks.
