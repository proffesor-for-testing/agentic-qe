# v3.9.1 Release Notes

**Release Date:** 2026-04-05

## Highlights

RVF persistent vector storage cluster (6 ADRs) replaces in-memory HNSW rebuild with native persistent indexes, adds per-agent COW memory branching, and unifies three HNSW implementations into one. StrongDM software factory and RuVector advanced capabilities round out the release.

## Added

- **RVF persistent vector storage (ADR-065/066)** — `RvfPatternStore` backed by @ruvector/rvf-node native HNSW. Cold-start drops from seconds to <2ms; search p50 under 0.2ms. Automatic fallback to SQLite when native unavailable.
- **Agent memory branching (ADR-067)** — Per-agent COW isolation via `derive()` with ingest-log merge on completion and automatic cleanup on failure. Wired into agent spawn/complete lifecycle.
- **Dream RVF COW (ADR-069)** — Dream engine uses RVF forks instead of SQLite savepoints. Recall validation gate compares top-k overlap before merge.
- **HNSW unification (ADR-071)** — Three HNSW implementations unified through `HnswLegacyBridge`. Shadow validator confirms <2% divergence. Old implementations decommissioned.
- **RVF dual-write migration (ADR-072)** — 5-stage migration adapter with consistency validator and stage gate enforcer. Stage 2 (dual-write, SQLite reads) active by default. MCP tools: `migration_status`, `migration_check`, `migration_promote`.
- **StrongDM software factory (ADR-062)** — Loop detection, holdout testing, gate ratcheting, progressive context, and meta-learning cycles across all 3 tiers.
- **RuVector advanced capabilities (ADR-087)** — EWC++ outcome recording, HDC fingerprinting, delta event sourcing, cognitive routing, and hyperbolic HNSW.
- **500+ new tests** across RVF, StrongDM, and RuVector capabilities.

## Fixed

- **`require()` broken in CLI bundles** — Added `createRequire` banner to both build scripts so all ESM chunks have a working `require`. Fixes RVF, sqlite, and native module paths that threw "Dynamic require of X is not supported".
- **`aqe init` path resolution** — Installers now resolve from package root via `findPackageRoot()`, fixing breakage under esbuild code-splitting chunk layouts.
- **`quality_assess` hangs on large projects** — Defaults to `src/` with `maxFiles=5000` cap and directory exclusions.
- **7 pre-existing test failures** — Fixed console spy targets, loop detection in MCP tests, dream cycle RVF paths, and coherence vector search fallback.

## Changed

- **@ruvector/gnn 0.1.19 -> 0.1.25** — Latest ProgressiveHnswBackend improvements.
- **3 RVF feature flags ship enabled** — `useRVFPatternStore`, `useAgentMemoryBranching`, `useUnifiedHnsw` default to `true`.
- **ESM-safe dynamic imports** in dream-engine and branch-manager.
