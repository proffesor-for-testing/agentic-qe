# v3.9.15 Release Notes

**Release Date:** 2026-04-22

## Highlights

qe-browser skill promoted to **Implemented** (ADR-091, trust tier 3). The skill's eval file is now a runnable evaluation via a new `aqe eval run --skill qe-browser` command, a CI workflow gates changes with unit + smoke + eval jobs, and Linux ARM64 users get a copy-paste `VIBIUM_BROWSER_PATH` hint after `aqe init --auto` detects their system chromium.

## Added

- **`aqe eval run --skill <skill>` CLI** — Executes a skill's `evals/<skill>.yaml` as real CommandRunner evaluations (not mocks). Uploads JSON results for post-run inspection. First consumer: qe-browser.
- **Linux ARM64 platform detection in `aqe init --auto`** — Probes `os.platform` + `os.arch` on Linux aarch64 and scans a prioritized list of system chromium locations. When a browser is found, prints an exact `export VIBIUM_BROWSER_PATH=...` line. When none exists, surfaces the `apt-get install chromium` remediation.
- **`docs/guides/qe-browser-getting-started.md`** — Install, smoke check, eval run, and example CLI invocations for all five primitives (`vibium assert`, `vibium batch`, `check-injection`, `visual-diff`, `vibium run-intent`).
- **`.github/workflows/test-qe-browser.yml`** — Three-job CI gate (vitest sweep, pinned-Vibium smoke test, eval run with JSON artifact upload). Fires on PRs touching the skill, its installer, or `CommandEvalRunner`.

## Fixed

- **qe-browser eval YAML no longer requires a local fixture server** — `tc006` / `tc007` previously pointed at `http://localhost:8088/qe-browser/SKILL.md.html`, which required an unstarted fixture server; rewrote both to use `https://httpbin.org/html`. `tc009` (poisoned-page check-injection) was dropped from the yaml with an explicit GAP comment — its severity logic is unit-tested at `tests/unit/scripts/qe-browser-check-injection.test.ts`.
- **Deterministic viewport in CI** — yaml `tc006` / `tc007` now include `vibium --headless viewport 1280 720` setup steps matching `smoke-test.sh`. No more "passes locally, fails in CI" from default-viewport differences.
- **Correct Vibium pin comment** — Previous comment claimed `^26.3.18` was a "major.minor-line pin"; under npm semver it accepts 26.99.x. Rewritten to accurately describe the intent: major-line pin blocking 27.0+ while allowing 26.x patches, with `smoke-test.sh` as the API-contract gate.

## Changed

- **ADR-091 status: Implemented (trust_tier 3)** — All gaps from the v3.9.9 introduction are closed. Dependent browser-using skills (`accessibility-testing`, `pentest-validation`, `enterprise-integration-testing`) now reference `qe-browser` as the canonical runner, with Playwright kept as a documented fallback where BiDi coverage is insufficient.

## Upgrade Notes

- `aqe init --auto` on Linux ARM64 hosts (Raspberry Pi, Apple Silicon Docker, AWS Graviton) now surfaces a `VIBIUM_BROWSER_PATH` export hint after Vibium install — copy the printed line into your shell before running `vibium` commands.
- Skill authors can ship an `evals/<skill>.yaml` file and run it with `aqe eval run --skill <skill>`. See `.claude/skills/qe-browser/evals/qe-browser.yaml` for a real example.
- No breaking changes. CLI surface additions only.
