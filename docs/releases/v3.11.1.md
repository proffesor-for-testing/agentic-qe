# v3.11.1 Release Notes

**Release Date:** 2026-06-25

## Highlights

Benchmark-driven deepening of v3.11.0's opt-in "cheap-first" test generation:
**best-of-k** and **cross-model** generation (measured ~+6 quality points), a
**self-test Goodhart guard**, a reusable **adversarial-verification gate**, **MCP
self-governance**, and **signed provenance** for delivered findings — all opt-in,
off by default. See the [Darwin-QE Self-Learning guide](../guides/darwin-qe-self-learning.md).

## Added

- **Best-of-k generation** (`freeTierBestOfK`) — try a few diverse candidates, keep
  the first that passes the objective check; extra cost only when the first fails.
- **Cross-model best-of-k** (`freeTierCandidateProviders`) — draw candidates from
  different model families so they cover each other's failures (measured ~+6 pts).
- **Cheap-first lane broadened** to requirements → BDD/Gherkin generation, with an
  off-topic relevance gate.
- **`@ruvector/adversarial-verify`** — host-agnostic blind-refuter verification +
  an opt-in `verifyGate` output gate. Export `agentic-qe/verification/adversarial-verify`.
- **MCP self-governance** — default-deny policy (`.harness/mcp-policy.json`) + CI gate.
- **Cost-Pareto value scoring** — rank models by quality-per-dollar from measured
  data. Export `agentic-qe/routing/value-score`.
- **Witnessed finding delivery** — Ed25519 hash-chained provenance, fail-closed verify.
- **Graceful optional-module loading** — native deps degrade instead of crashing, with
  a CI absent-drill that proves the degraded path without masking real failures.

## Changed

- **Default free-tier model → `qwen3:30b-a3b`** (was `qwen3:8b`): the 8B is below the
  test-generation quality floor; the 30B clears it (~89% mutation kill). Only affects
  users who opt into the free tier — set `freeTierModel` to fit your hardware.
- **Goodhart guard** — only an objective check (real run, coverage, mutation, schema)
  can lift routing confidence; a model's own self-authored test never does.

## Notes

This release is **fully additive and opt-in** — existing users see no behavior change
unless they enable the free tier. The honest record (including the negative results
we kept) lives in `docs/metaharness/06-darwin-qe-self-learning-action-lane.md` and
ADR-111.
