# v3.8.2 Release Notes

**Release Date:** 2026-03-17

## Highlights

Bug fix release addressing 8 issues reported by Nate during real-world testing of `aqe init` on macOS with global npm install. Also completes the ruflo rebrand and adds a validation pipeline helper.

## Fixed

- **YAML frontmatter validation** — 114 SKILL.md files had blank lines inside frontmatter that broke parsers (#360)
- **OpenCode installer paths** — Global npm installs now correctly find bundled agents/skills/tools instead of looking in the user's CWD (#361)
- **Settings permissions overwrite** — `aqe init --upgrade` preserves user-added permission entries via union-merge (#362)
- **Hook path fallback** — Brain-checkpoint and statusline hooks use hardcoded project root fallback instead of unreliable `pwd` (#363)
- **Agent updated date** — Overlay-modified agents now get their `updated:` frontmatter field bumped (#365)
- **Validation pipeline docs** — `--steps` parameter consistently documented as comma-separated (#367)

## Added

- **Validation pipeline helper** — `.claude/helpers/validation-pipeline.cjs` with 13 requirements validation steps, gate enforcement, per-step scoring, and weighted rollup (#364)

## Changed

- **Security scanner latency** — `qe-dependency-mapper` dependency downgraded from `hard` to `soft`, eliminating wait time for simple scans (#366)
- **Ruflo rebrand complete** — All remaining `@claude-flow/cli` references replaced with `ruflo` across skills, helpers, and bridge modules
- **CI stability** — Resolved test timeouts and cancellations (#350)
