# v3.9.33 Release Notes

**Release Date:** 2026-05-18

## Highlights

A dependency-cleanup and supply-chain-security release.
`npm install agentic-qe` now pulls **~480 fewer transitive packages**,
ships with **zero high/critical CVEs in the consumer install** (was 5 in
v3.9.32 — masked from in-repo audit by root-only `overrides`), and emits
**no `ERESOLVE` peer warnings**. Two new CI gates make future regressions
of this class impossible to ship.

End-to-end verification in a clean tmp project:

| Signal | v3.9.32 | v3.9.33 |
|---|---|---|
| Packages installed | ~964 | 183 |
| `ERESOLVE` warnings | many | 0 |
| Deprecation warnings | 5+ | 2 (both transitive of native chains) |
| Consumer-side CVEs | 4 high + 1 critical | **0** |
| `aqe init --auto` | green | green (93 ms) |

## Fixed

- **Critical CVE chain eliminated.** `@xenova/transformers@2.17.2` (last
  publish 2024-05-29) pinned `onnxruntime-web@1.14.0`, which pulled
  `onnx-proto` → `protobufjs <=7.5.5`. That chain has 8 GHSA advisories
  including a critical arbitrary-code-execution one
  ([GHSA-xq3m-2v4x-88gg](https://github.com/advisories/GHSA-xq3m-2v4x-88gg)).
  Our top-level `overrides` block hid the vulns from in-repo
  `npm audit` (overrides are root-only by design), but every user
  installing `agentic-qe` was getting them. Migrated to
  `@huggingface/transformers@4.2.0` — the official maintained successor.
  It depends on `protobufjs ^7.2.4` directly with no `onnx-proto`
  middleman, so the version range resolves cleanly to the patched line.
  End-user `npm audit` now reports **0 vulnerabilities**.

- **Self-learning consolidation `lastUsedAt` parsing** ([#493](https://github.com/proffesor-for-testing/agentic-qe/issues/493)).
  Bridge-mining and pattern consolidation rehydrate `Date` fields at all
  kv-read seams, so the loop no longer silently drops freshly written
  experiences whose timestamps came back as strings from the persistence
  layer.

- **6 high-severity CVEs in `@typescript-eslint` v6 chain.** The
  `@typescript-eslint/typescript-estree` → `minimatch` chain had 6
  high-severity advisories. Bumped the dev-side toolchain to
  `@typescript-eslint@8.59.3`.

## Changed

- **`@claude-flow/browser` and `@claude-flow/guidance` are now optional
  peer deps** (was `optionalDependencies`). Together these no longer
  pull the internally-inconsistent `@claude-flow/cli@3.6.x` transitive
  chain that was the source of all `ERESOLVE` warnings (its child
  `@claude-flow/embeddings` declares peers that disagree with the
  parent CLI's pins). Tree shrinkage: **482 transitive packages
  removed**, plus the long tail of deprecated transitives
  (`koa-router@14`, `node-domexception@1`, `uuid@9`).

  Users who want either feature install explicitly:
  ```bash
  npm install -g @claude-flow/browser@3.0.0-alpha.1
  npm install -g @claude-flow/guidance@3.0.0-alpha.1
  ```
  And `aqe init` now tells you this — see Added.

- `web-tree-sitter` 0.24 → 0.26. The `Parser.init` API moved onto the
  `Parser` class. A small compatibility shim in
  `tree-sitter-wasm-parser.ts` makes us work against both surfaces.

- Toolchain majors: `@typescript-eslint/{eslint-plugin,parser}` 6 → 8,
  `commander` 12 → 14, `esbuild` 0.27 → 0.28, `@types/node` 20 → 25.

- Lockfile refreshed within caret: `vitest` 4.1.4 → 4.1.6,
  `@vitest/coverage-v8` 4.1.4 → 4.1.6, `axe-core` 4.11.3 → 4.11.4,
  `better-sqlite3` 12.9.0 → 12.10.0, `jose` 6.2.2 → 6.2.3,
  `msw` 2.13.4 → 2.14.6, `ora` 9.3.0 → 9.4.0, `tsx` 4.21.0 → 4.22.1,
  `yaml` 2.8.3 → 2.9.0.

## Added

- **Optional add-ons hint in `aqe init`.** After a successful init, when
  `@claude-flow/browser` or `@claude-flow/guidance` is not installed
  locally, the CLI appends a one-line install command for each — so
  opting in doesn't require reading the README. Skipped silently when
  both are present. Detection uses `require.resolve` and correctly
  treats `ERR_PACKAGE_PATH_NOT_EXPORTED` as "installed" for packages
  that only ship an ESM export condition.

- **`security-audit` job in CI** (`optimized-ci.yml`). Blocks PRs on
  high/critical CVEs in production deps (`npm audit --audit-level=high
  --omit=dev`); warns on dev-dep CVEs. The same gate runs as a
  pre-publish step in `npm-publish.yml`.

- **Consumer-side audit gate** (`consumer-audit` job in
  `npm-publish.yml`). Packs the local build, installs the tarball into
  a fresh tmp project, and runs `npm audit` *there* — closing the gap
  where root-only `overrides` mask consumer-facing vulns from in-repo
  audit. Now a `needs:` of the publish job. This gate would have caught
  the `protobufjs` chain the moment we shipped v3.9.32; from now on,
  consumer-facing CVEs cannot ship undetected.

- **Post-publish consumer audit** (`post-publish-canary.yml`). Same
  audit pattern against the freshly-published registry tarball as a
  non-blocking safety net. Findings open an auto-labelled P1 issue
  (`security`, `canary`, `release`).

## Behind the scenes

The peer-dep mismatch inside `@claude-flow/cli@3.6.x` that triggered all
the `ERESOLVE` warnings is a real upstream bug — `@claude-flow/cli` pins
`@claude-flow/embeddings@^3.0.0-alpha.12` as an optional dep, but
`embeddings` declares peers on `@ruvector/attention@^2.2.2` and
`agentic-flow@^2.0.0` while the parent CLI pins `^0.1.32` and
`^3.0.0-alpha.1`. We work around it by no longer pulling the chain at
all (unless the user opts into the optional peers). A proper fix
requires upstream coordination on `@claude-flow/cli`.
