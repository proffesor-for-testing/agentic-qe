# v3.12.1 Release Notes

**Release Date:** 2026-07-12

## Highlights

`aqe init` is now non-destructive to configuration you already have. Running it
in a project that already has a `.claude/settings.json` (for example after
`ruflo init`) no longer overwrites your settings — it merges, preserves your
values, and takes a one-time backup of the original file, just like `CLAUDE.md`.

## Added

- **`.claude/settings.json.backup`** — `aqe init` writes a one-time backup of the
  pristine original `settings.json` before modifying it, mirroring the existing
  `CLAUDE.md.backup` behavior. The backup is never overwritten on subsequent
  re-inits, so your original settings are always recoverable.

## Fixed

- **Init no longer strips ruflo / Claude Flow hooks.** AQE's hook-detection was too
  broad (it matched anything under `.claude/helpers/` or any `ruflo` command), so
  re-running init in a project that had already run `ruflo init` deleted the user's
  hooks. Detection is now scoped to AQE's own hook markers; other tools' hooks are
  preserved.
- **Init preserves a custom `statusLine`.** It only writes its own status line when
  none exists or the existing one is AQE's.
- **Init respects an explicit `includeCoAuthoredBy` setting.** The default is only
  written when you haven't chosen a value.
- **Init preserves user overrides of `AQE_*` environment variables.** Re-init fills
  in only the missing keys; values you changed (e.g. `AQE_LEARNING_ENABLED=false`)
  are kept.
- **`v3Configuration` / `v3Learning` sections are deep-merged**, so any keys you add
  in those sections survive re-init instead of being replaced wholesale.

## Changed

- Permissions, hooks, MCP servers, and env in `.claude/settings.json` are now all
  merged additively — `aqe init` only adds AQE's own entries and never removes
  entries it doesn't own.
