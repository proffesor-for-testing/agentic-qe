# v3.8.3 Release Notes

**Release Date:** 2026-03-18

## Highlights

Major skill system overhaul based on Anthropic's skill design guidance — all 84 QE skills restructured with gotchas, progressive disclosure, composition, and 5 new skills. Plus 7 bug fixes for first-run experience (portable hooks, OpenCode integration, MCP startup errors).

## Added

- **ADR-086 skill design standards** — All 84 QE skills restructured as folder-based systems with progressive disclosure, gotchas from production learning data, config, and run history. Based on Anthropic's "Lessons from Building Claude Code" article.
- **5 new skills** — `test-failure-investigator`, `coverage-drop-investigator`, `e2e-flow-verifier`, `test-metrics-dashboard`, `skill-stats`
- **5 on-demand hook skills** — `strict-tdd`, `no-skip`, `coverage-guard`, `freeze-tests`, `security-watch` with executable scripts
- **Gotchas sections** — 30 skills now include battle-tested failure data from production learning records
- **Reference and template files** — 10 files across 7 skills (OWASP, k6, mutation operators, security reports, etc.)
- **Skill composition** — Cross-references across 10 key skills for chaining workflows
- **Config and run-history** — `config.json` with `_setupPrompt` for 7 skills; `run-history.json` for 5 metric-producing skills

## Fixed

- **Portable hook commands** — `aqe init` no longer embeds the host machine's absolute path into hook commands, making `.claude/settings.json` portable across machines. ([#369](https://github.com/proffesor-for-testing/agentic-qe/issues/369))
- **OpenCode config schema** — `aqe init --with-opencode` now generates valid `opencode.json` matching OpenCode's `McpLocal.strict()` schema. ([#370](https://github.com/proffesor-for-testing/agentic-qe/issues/370))
- **Infra-healing fallback playbook** — Fixed field name mismatches that caused a TypeError on every MCP startup. ([#371](https://github.com/proffesor-for-testing/agentic-qe/issues/371))
- **Fleet init crash** — `LearningOptimizationCoordinator` now gracefully degrades when SONA init fails. Combined with the circular init fix, fleet initialization succeeds on MCP startup. ([#372](https://github.com/proffesor-for-testing/agentic-qe/issues/372))
- **SONA persistence initialization** — Fixed circular `ensureInitialized()` call during `_doInitialize()` when `useSONAThreeLoop` flag is enabled.
- **RuVector feature flag test compatibility** — 40+ test failures resolved from v3.8.0 default-flags-on change.
- **CI stability** — Resolved CI workflow timeouts and broken contract test path. ([#350](https://github.com/proffesor-for-testing/agentic-qe/issues/350))

## Changed

- **Skill descriptions rewritten** — All 84 skills now use "Use when..." trigger conditions for better auto-detection
- **Textbook knowledge stripped** — 892 lines removed from 15 skills (26% size reduction)
- **3 redundant skills removed** — `qe-contract-testing`, `qe-security-compliance`, `aqe-v2-v3-migration`. `aqe init --upgrade` auto-cleans these.

## Install / Upgrade

```bash
npm install -g agentic-qe@3.8.3
cd your-project && aqe init --auto
```
