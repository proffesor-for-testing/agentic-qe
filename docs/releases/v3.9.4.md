# v3.9.4 Release Notes

**Release Date:** 2026-04-06

## Highlights

v3.9.4 is a reliability hotfix for two issues the v3.9.3 release did not catch:

1. **Governance phase failing** on all chunked-bundle installs (observed in cf-devpod) — same `__dirname + '../..'` chunk-split regression pattern that v3.9.3 Fix 4 repaired for `findMcpEntry()`. I missed the sibling installer.
2. **Phase 06 `code-intelligence pre-scan` hang** on codespace environments (observed in ruview) — v3.9.3's `Promise.race`+`setTimeout` watchdog cannot interrupt synchronous native code that blocks the Node event loop. That's a structural limitation, not a bug we can patch in place.

v3.9.4 ships targeted fixes for (1) and two mitigations for (2): an escape hatch so users can unblock immediately, and per-file progress logging so the stall source becomes visible.

The **proper** fix for (2) — running the indexer in a killable worker thread so the watchdog works regardless of native-code blocking — is tracked for v3.9.5 in [issue TODO](https://github.com/proffesor-for-testing/agentic-qe/issues/TODO).

## Upgrade

```bash
# Claude Code
claude mcp add agentic-qe -- npx -y agentic-qe@3.9.4
aqe init --auto

# Global
npm install -g agentic-qe@3.9.4
aqe init --auto
```

If `aqe init --auto` stalls in phase 06 on your project, unblock with:

```bash
AQE_SKIP_CODE_INDEX=1 aqe init --auto
# or
aqe init --auto --skip-code-index
```

Then report the last `[N/M] path/to/file.ts` line you see before the stall to [issue TODO](https://github.com/proffesor-for-testing/agentic-qe/issues/TODO) so we can target the underlying native cause.

## Fixed

- **Governance phase failure on chunked bundles** — `getGovernanceAssetsPath()` in `src/init/governance-installer.ts` now uses `findPackageRoot(import.meta.url)` walk-up instead of fixed `__dirname + '../..'` traversals. Same fix applied to `kiro-installer.ts` (`--with-kiro` opt-in path) and `src/cli/commands/sync.ts` (`aqe sync init` command). All three were on the same broken pattern that skills/agents/opencode/n8n installers already migrated away from in v3.9.1.

- **Phase 06 hang with no diagnostic output** — the existing watchdog cannot help when a native call blocks the event loop synchronously, but users deserve visibility into what the indexer is doing. Per-file logging now shows every file before it starts, so the last log line names the exact file that stalls.

## Added

- **`AQE_SKIP_CODE_INDEX=1` env var** — bypasses phase 06 entirely. Useful for CI pipelines and for users whose project hits the phase 06 native stall. Lazy on-demand indexing via `aqe code index` still works.

- **`--skip-code-index` CLI flag** — interactive equivalent of the env var. Works with `--auto`, wizard, and upgrade modes.

- **Per-file progress log in phase 06** — format `[current/total] relative/path/to/file.ts` emitted before each file is processed. Replaces the previous "every 100 files" summary.

## Verification

| Scenario | v3.9.3 | v3.9.4 |
|---|---|---|
| Fresh init, 1-file fixture | 385 ms, `x Install governance configuration (1ms)` | **502 ms, `* Install governance configuration (4ms)`** |
| `aqe init --auto --skip-code-index` | `error: unknown option` | **209 ms, phase 06 skipped cleanly** |
| `AQE_SKIP_CODE_INDEX=1 aqe init --auto` | phase 06 runs normally | **phase 06 skipped with explanatory log** |
| Per-file progress log | every 100 files only | **every file — `[1/349] src/...`** |

## Known Issues / Deferred

- **Phase 06 native-call stall (ruview)** — root cause still unknown. v3.9.4's per-file log will expose the exact file that stalls on the next run. The permanent fix (worker thread isolation with terminate-on-timeout) is scheduled for v3.9.5.

- **Pre-existing approximate-HNSW test assertion flake** — tracked in #399, unchanged from v3.9.3.

## What's NOT in this release

- **No architectural change to the indexer.** The worker-thread refactor is larger-scope and needs careful review. v3.9.4 is a same-day hotfix to unblock users; the proper fix comes next.
- **No feature flag defaults changed.** All v3.9.1 feature benefits (RVF pattern store, unified HNSW, agent memory branching, KG semantic search) remain on.

## User-perspective verification

If you upgrade to v3.9.4 and want to confirm both fixes:

```bash
# (1) governance phase no longer fails
cd /tmp && rm -rf aqe-verify && mkdir aqe-verify && cd aqe-verify
echo "export const x = 1;" > file.ts
aqe init --auto 2>&1 | grep "governance"
# Expected: * Install governance configuration (Nms)  — with asterisk, not x

# (2) per-file logging is visible
aqe init --auto 2>&1 | grep "\[1/"
# Expected: a line like "  [1/1] file.ts"

# (3) escape hatch works
rm -rf .agentic-qe .claude .mcp.json CLAUDE.md
AQE_SKIP_CODE_INDEX=1 aqe init --auto 2>&1 | grep "skipped"
# Expected: "Code intelligence skipped (AQE_SKIP_CODE_INDEX=1)"
```
