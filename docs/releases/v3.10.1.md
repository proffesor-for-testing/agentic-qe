# v3.10.1 Release Notes

**Release Date:** 2026-05-22

## Highlights

The **ADR-043 LLM router is now actually wired** into the production swarm
path. HybridRouter (ADR-043) and per-service `analyzeXxxWithLLM()` branches
(ADR-051) were implemented in previous releases but never injected — every
domain service accepted `llmRouter` as an optional dependency and zero
callers passed one, so `isLLMAnalysisAvailable()` returned `false` everywhere
and the LLM branches in 15 service paths across 11 domains were unreachable.
Fleets booted with any provider API key in env will now route LLM-enhanced
analysis through that provider; previously the same fleet ran zero LLM calls
from the swarm path regardless of provider config. Includes persistent
`aqe llm config --set`, Gemini `GOOGLE_API_KEY` alias, two real
HybridRouter fallback-chain bug fixes surfaced by an E2E pass, and a
qe-devils-advocate audit hardening sweep that closed 10 findings before
merge.

## Changed — behavioral

- **Kernel constructs a `HybridRouter` singleton during `initialize()`** and
  forwards it through every domain factory to its services. Auto-enabled
  when any provider API key (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`,
  `OPENROUTER_API_KEY`, `GOOGLE_AI_API_KEY`, `GEMINI_API_KEY`,
  `GOOGLE_API_KEY`) is in env or when `.agentic-qe/llm-config.json` explicitly
  enables a provider.
- **MCP standalone tools resolve the same singleton** via
  `getSharedLLMRouter()` so kernel-mode and MCP-mode share one router
  instance — single cost tracker, cache, circuit breaker.
- **Three opt-out mechanisms**, in order of preference:
  1. `AQE_LLM_ROUTER_DISABLED=1` env var (no code change required)
  2. `KernelConfig.llmRouter: { enabled: false }` (programmatic)
  3. Unset the relevant provider keys in env
- **`aqe llm config --set` now persists.** Writes go to
  `.agentic-qe/llm-config.json` and survive a process restart. The file is
  apiKey-stripped on write so secrets can't end up in version control.
  `aqe llm config --set apiKey=...` now errors loudly and points at the env
  vars users should use instead.
- **Gemini provider accepts `GOOGLE_API_KEY`** in addition to
  `GOOGLE_AI_API_KEY` and `GEMINI_API_KEY` — covers users with the
  conventional Google API key name without requiring an env alias.
- **Init observability:** kernel publishes
  `kernel.llm-router.init-failed` and `kernel.llm-router.init-no-provider`
  events on the event bus when router construction fails or no provider
  is available. Monitoring can subscribe to catch silent regressions.

## Fixed

- **HybridRouter fallback chain was provider-discriminatory.**
  `executeWithFallback` previously hardcoded `['claude', 'openai',
  'ollama']` as the only fallback providers; users with only
  `GEMINI_API_KEY` (or openrouter / azure / bedrock) had no fallback when
  the routing rule's chosen provider was unavailable. Now includes the
  user's `defaultProvider` automatically AND honors all 7
  ExtendedProviderType providers in the fallback chain.
- **Non-retryable error from one provider short-circuited the entire
  fallback loop.** A non-retryable failure (e.g., Claude "invalid
  x-api-key") would fail the request even when other providers were
  available. Non-retryable errors now skip the retry-delay but continue to
  the next fallback entry.

## Audit hardening

A qe-devils-advocate audit on 2026-05-21 surfaced 10 issues
post-initial-wiring; all addressed before merge:

1. Silent billing regression (router instantiated but not threaded)
2. Two unwired MCP tools
3. Decorative `MCPToolContext` field with no resolver
4. Split router singletons (MCP vs kernel)
5. Silent error swallowing in init
6. Two real HybridRouter routing bugs (fallback chain + non-retryable short-circuit)
7. Missing behavioral tests
8. Miscounted services
9. Silent apiKey drops on `--set`
10. Undocumented env precedence

See ADR-043 "Audit findings" section for details.

## Migration

No migration required. Default behavior preserved for fleets with no
provider keys in env (router is off, no behavior change).

To enable the LLM router on a previously-deployed fleet:

```bash
# Pick a provider — any of these will auto-enable the router on next boot:
export ANTHROPIC_API_KEY=...   # or
export OPENAI_API_KEY=...      # or
export GOOGLE_API_KEY=...      # or
export OPENROUTER_API_KEY=...
```

To explicitly opt out even with keys in env:

```bash
export AQE_LLM_ROUTER_DISABLED=1
```
