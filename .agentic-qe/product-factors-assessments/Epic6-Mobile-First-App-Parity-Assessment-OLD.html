‚ùì Clarifying Questions

These questions help uncover "unknown unknowns" and refine requirements before development.

Structure

Q1: What Service Worker caching strategy will be used (Cache First, Network First, Stale While Revalidate)? Different strategies for different content types?
Q2: Which specific barcode formats need to be supported (UPC-A, EAN-13, QR codes, others)? Any custom internal barcode formats?
Q3: Will Workbox library be used for Service Worker management, or custom implementation? If custom, what's the fallback strategy?

Function

Q4: What is the minimum engagement criteria for A2HS prompt (2 visits? Time spent? Specific actions completed)?
Q5: Which specific features should work offline (browse products, view cart, search, checkout)? What's the priority order?
Q6: How should the system handle checkout attempts while offline (queue for Background Sync, block with error, save draft)?
Q7: What happens if biometric authentication is not available (device doesn't support, user disabled)? Mandatory fallback to password?
Q8: For "store mode", what specific features are included (scan, check stock, store map, employee assistance request)?
Q9: What content should be pre-cached for offline (homepage, top 100 products, full catalog)? Size constraints?

Data

Q10: What is the maximum cache size allowed per user (50MB iOS limit, 10% disk on Firefox)? How to handle quota exceeded?
Q11: How long should cached content remain valid before requiring refresh (24 hours, 7 days, until next online session)?
Q12: Should cart items be synced across devices in real-time or only on demand (user triggers sync)?
Q13: What data should persist in IndexedDB vs LocalStorage (products in IDB, preferences in LS)? Clear partitioning strategy?

Interfaces

Q14: What is the exact integration flow with Apple Pay merchant account (payment processor, tokenization provider)?
Q15: Which push notification provider will be used (Firebase Cloud Messaging, OneSignal, custom)? Rate limits?
Q16: How should cross-device session sync work technically (backend API sync, cloud storage, peer-to-peer)?
Q17: Are there specific UI/UX requirements for the PWA install prompt (custom vs browser default)?

Platform

Q18: What is the minimum browser version support (Chrome 80+, Safari 13+, Firefox 75+)? Polyfills for older versions?
Q19: Should PWA work on desktop browsers (Chrome, Edge, Safari) or mobile-only? Different UX for desktop PWA?
Q20: What's the strategy for iOS limitations (50MB cache, no Background Sync, limited push notifications)?
Q21: How should the app handle devices without biometric sensors (password-only login acceptable)?

Operations

Q22: What user analytics should be tracked for PWA usage (install rate, offline sessions, express checkout conversion)?
Q23: How should the app handle users who uninstall PWA but keep using mobile web? Retain cache?
Q24: What accessibility support is required beyond WCAG 2.2 (screen reader testing on iOS/Android, voice control)?
Q25: Should barcode scanning work in both portrait and landscape orientations? Auto-rotate handling?

Time

Q26: What's the retry strategy for Background Sync (exponential backoff, max retries, give up after how long)?
Q27: How frequently should push notifications be sent for price drops (immediate, daily digest, weekly)? User customizable?
Q28: What's the Service Worker update strategy (update on refresh, update in background, notify user)?
Q29: How long should pending offline actions be retained (7 days, 30 days, indefinitely until online)?

Success Metrics

Q30: How is "Mobile Conversion Rate" calculated (mobile web only, PWA only, combined)? Baseline measurement?
Q31: What constitutes a "PWA install" for the 10% target (A2HS completion, first launch after install, 7-day retention)?
Q32: Are there specific performance budgets for PWA (First Contentful Paint < 2s, Time to Interactive < 5s on Slow 3G)?
