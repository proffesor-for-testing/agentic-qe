Remediation: Move token to n8n credential store. Use environment variables or secrets management.

HIGH SEC-002: Missing Input Validation

Location: Multiple Code nodes

Description: AI agent outputs are parsed as JSON without validation. Malicious prompt injection could lead to code execution or data exfiltration.

Affected Nodes: Cleanup Output to JSON, Cleanup Code, Code Cleanup CMO

CVSS Score: 7.5 (High) | CWE: CWE-20 Improper Input Validation

The webhook trigger has no rate limiting configured. An attacker could trigger excessive workflow executions, leading to resource exhaustion or API quota depletion.

// Analyzed: JSON extraction from AI markdown output
// Pattern: Regex-based extraction with fallback

const markdownMatch = rawText.match(/```json\s*([\s\S]*?)\s*```/);
if (markdownMatch) {
    jsonString = markdownMatch[1];
} else {
    // Fallback: Find first { to last }
    const start = rawText.indexOf('{');
    const end = rawText.lastIndexOf('}');
    jsonString = rawText.substring(start, end + 1);
}

// Issue: No validation of JSON structure before parse
return { json: JSON.parse(jsonString) };

WARNING Missing try/catch around JSON.parse(). If AI returns malformed JSON, this will throw an unhandled exception.

2. Format Output (Meta Ads)
// Analyzed: Markdown table generation

// Issue: NaN propagation if spend contains non-numeric string

WARNING parseFloat("invalid") returns NaN. NaN + number = NaN. Total spend could become NaN if any spend value is malformed.
